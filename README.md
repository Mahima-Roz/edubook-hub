
📚 EduBook Hub
Project Overview

EduBook Hub is a web-based knowledge sharing platform designed to improve book accessibility among students in Bangladesh. The system enables users to donate, borrow, or purchase books at minimal cost while reducing book wastage. The platform encourages reading habits through rewards, premium membership, and community events while ensuring secure, system-managed interactions between users.

⭐ Key Features

User registration with general and premium membership

Book donation, lending, and borrowing system

Map-based search to find the nearest available books

User dashboard displaying borrowed and lent books

Admin verification system for book quality control

Reward badges to encourage reading habits

Premium membership with payment integration (bKash/Nagad)

Community features such as book festivals and local events

⚙ Software Development Methodology

The project follows the Agile Scrum model for iterative and incremental development.

Reasons for choosing Scrum:

Supports frequent requirement changes

Allows incremental delivery of features

Encourages continuous user feedback

Works well with small development teams

🔄 Git Workflow

The project followed a structured Git collaboration workflow:

Step 1 – Clone Repository

Developers cloned the main repository from GitHub.

Step 2 – Create Feature Branch

Each developer created a separate branch for their feature.

Example:

feature-khalil
feature-mahima
feature-nishad
feature-mahfuz
Step 3 – Implement Features

Each team member developed their assigned modules independently.

Example responsibilities:

UI/UX design

Backend setup

Competitive analysis

Functional verification

Step 4 – Commit Changes

Developers committed their changes locally with descriptive commit messages.

Step 5 – Push to GitHub

Branches were pushed to the remote repository.

Step 6 – Pull Request

Developers created Pull Requests to merge their feature branch into the main branch.

Step 7 – Merge to Main Branch

After review, the code was merged into the main branch.
